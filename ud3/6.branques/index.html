    6. Treballant amb branques

    Branques

    Quan hem de fer canvis al nostre codi que no siguen immediats, ja siga perquè volem corregir errors, o per afegir codi nou que pot ser inestable, el més pràctic és utilitzar branques. Recordem que una branca és una bifurcació del codi principal que ens permet seguir un camí de desenvolupament diferent, i bé descartar els canvis realitzats o fusionar-los amb la branca principal.

    Tot i que pràcticament tots els sistemes de control de versions ofereixen mecanismes per treballar amb branques, la manera com Git les gestiona fa que sobreisca sobre la resta, sent extremadament ràpid i senzill realitzar el canvi.

    Vegem-ne un petit exemple de funcionament:

    Com podem veure, hi ha tres confirmacions a la branca principal (master). A partir d'aquest tercer commit, es crea la branca de proves, a la qual s'afegeixen tres commits més, corresponents a canvis no definitius al codi. Finalment, una vegada validats aquests canvis i comprovat-ne el funcionament, es fusionen amb el contingut de la branca Màster.

    Hem de tenir en compte, que a cada moment, el HEAD apuntarà a l'últim commit realitzat.

    Treballant amb branques

    Anem a fer un cas pràctic on crearem un nou repositori per treballar amb branques. És important que recordeu que una branca era una bifurcació de la branca principal de desenvolupament que ens permet seguir treballant en noves funcionalitats per a les nostres aplicacions o fent proves sense necessitat de modificar la branca principal.

    Preparació del repositori

    Anem a partir d'una carpeta per a un projecte sota control de versions. Per preparar aquest escenari, fem els passos següents:

    • Pas 1. Creem la carpeta i inicialitzem el repositori
    $ mkdir projecte2
$ cd proyecto2/
$ git init
Initialized empty Git repository in /home/dam/scv/projecto2/.git/
    • Pas 2. Creem contingut al repositori

    Ara crearem un fitxer i afegirem contingut. Farem dues confirmacions, una per operació:

    $ touch fitxer1.md
$ git add fitxer1.md
$ git commit -a -m "Afegit fitxer1.md"

[master (root-commit) 63d1a06] Afegit fitxer1.md
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 fitxer1.md

$ echo "# Text de prova 1" > fitxer1.md
$ git commit -a -m "Modificat fitxer1.md"

[master 5513ed9] Modificat fitxer1.md
 1 file changed, 1 insertion(+)

    Veiem el registre de git:

    $ git log --oneline
5513ed9 (HEAD -> master) Modificat fitxer1.md
63d1a06 Afegit fitxer1.md
    • Pas 3. Creació d'una branca de proves

    Crearem ara una branca que anomenarem proves. Per això farem servir l'ordre git branch:

    $ git branch proves

    Per consultar les branques disponibles fem servir la mateixa ordre, sense especificar el nom:

    $ git branch
* màster
 proves

    L'ordre git branch ens mostra les branques existents, i indica amb un asterisc a quina branca ens trobem.

    • Pas 4. Treballem a la branca de proves

    Per passar a una branca concreta, farem servir git checkout:

    $ git checkout proves

Switched to branch 'proves'

$ git branch
 màster
* proves

    Com veiem, ara estem a la branca proves. Crearem un fitxer que anomenarem prova1.md:

    $ touch prova1.md
$ git add prova1.md
$ git commit -m "Afegit prova1.md"

[proves 35d135f] Afegit prova1.md
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 prova1.md

    Si ara observem el log de git:

    $ git log --oneline
35d135f (HEAD -> proves) Afegit prova1.md
5513ed9 (master) Modificat fitxer1.md
63d1a06 Afegit fitxer1.md

    Veiem com la branca màster es va quedar en un commit anterior i que el HEAD apunta a aquest últim commit a la branca de proves.

    • Pas 5. Saltant entre branques

    Si volem tornar a la branca principal, haurem d'utilitzar novament l'ordre git checkout. Vegem abans un detall important. Si llistem el contingut del directori actual:

    $ ls
fitxer1.md prova1.md

    Veiem que tenim el fitxer1.md creat a la branca principal abans de fer la bifurcació i el fitxer prova1.md generat en aquesta branca.

    Si tornem a la branca principal i observem el contingut:

    $ git checkout master
Switched to branch 'master'

$ ls
fitxer1.md

    Com veiem, només disposem del fitxer.md que és el contingut que es correspon a la branca master. Observeu que és git qui s'encarrega a través de l'ordre checkout, de portar el contingut de cadascuna de les branques al directori de treball.

    Si ara, dins de la branca master consultem el registre de canvis:

    $ git log --oneline
5513ed9 (HEAD -> master) Modificat fitxer1.md
63d1a06 Afegit fitxer1.md

    Veiem que apareix el log de la branca màster, que és la branca en què estem treballant en aquests moments.

    • Pas 6. Combinant branques

    Una vegada hem finalitzat les proves o el desenvolupament experimental que va provocar la bifucació del projecte tenim dues opcions. Bé descartar aquesta branca perquè la funcionalitat o les proves no són les esperades, o bé combinar aquesta branca amb la principal.

    Per combinar el contingut de la branca proves amb la branca màster principal, hem d'utilitzar l'ordre git merge de la manera següent.

    Des de la branca màster del nostre projecte:

    $ git merge proves
Updating 5513ed9..35d135f
Fast-forward
 prova1.md | 0
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 prova.md
 ```

Com ens indica, actualitza el commit amb *SHA-1 5513ed9* amb el commit amb el *SHA-1 35d135f*, on s'ha afegit el fitxer `prova1.md`.

Si analitzem ara el registre:

```bash
$ git log --oneline

35d135f (HEAD -> master, proves) Afegit prova1.md
5513ed9 Modificat fitxer1.md
63d1a06 Afegit fitxer1.md

    Com veiem, el HEAD apunta a l'últim commit, tant a la branca màster com a proves, indicant que totes dues estan sincronitzades. Ara bé, amb el merge, el que hem fet és portar el contingut de la branca proves a la branca màster, però això no elimina la branca. Això ens permet disposar d'una branca principal, amb les versions estables i provades dels nostres programes, i una altra branca de desenvolupament, sobre la qual treballarem els canvis futurs de l'aplicació, i que combinarem amb la principal un cop estiguin aquests canvis finalitzats i provats.

    • Pas 7. Descartant els canvis d'una branca

    Quan el resultat del desenvolupament en una branca no és satisfactori i volem descartar-lo, farem servir l'ordre git reset.

    Per exemple, si ara a la branca de proves anterior hauríem afegit alguns fitxers més en diversos commits:

    $ git checkout proves

Switched to branch 'proves'

$ echo "Contingut a descartar 1" > fitxer_a_descartar_1.md

$ git add fitxer_a_descartar_1.md

$ git commit -m "Afegit fitxer_a_descartar_1.md."

[proves f91a296] Afegit fitxer_a_descartar_1.md.
 1 file changed, 1 insertion(+)
 create mode 100644 fitxer_a_descartar_1.md

$ echo "Contingut a descartar 2" > fitxer_a_descartar_2.md

$ git add fitxer_a_descartar_2.md

$ git commit -m "Afegit fitxer_a_descartar_2.md."

[proves 819d75e] Afegit fitxer_a_descartar_2.md.
 1 file changed, 1 insertion(+)
 create mode 100644 fitxer_a_descartar_2.md

$ echo "Contingut a descartar 3" > fitxer_a_descartar_3.md

$ git add fitxer_a_descartar_3.md

$ git commit -m "Afegit fitxer_a_descartar_3.md."

[proves 0d27dd0] Afegit fitxer_a_descartar_3.md.
 1 file changed, 1 insertion(+)
 create mode 100644 fitxer_a_descartar_3.md

    El log de git queda de la manera següent:

    $ git log --oneline

0d27dd0 (HEAD -> proves) Afegit fitxer_a_descartar_3.md.
819d75e Afegit fitxer_a_descartar_2.md.
f91a296 Afegit fitxer_a_descartar_1.md.
35d135f (master) Afegit prova1.md
5513ed9 Modificat fitxer1.md
63d1a06 Afegit fitxer1.md

    Si el que desitgem ara és descartar els canvis realitzats i tornar al punt on vam fer la bifurcació de màster (identificador SHA-1 35d135f), ho podem fer amb:

    $ git reset --hard 35d135f

HEAD is now at 35d135f Afegit prova1.md

    Amb això s'han descartat tots els canvis realitzats a partir del commit indicat:

    $ git log --oneline

35d135f (HEAD -> master, proves) Afegit prova1.md
5513ed9 Modificat fitxer1.md
63d1a06 Afegit fitxer1.md

    Hem de destacar que aquesta operació de reset pot realitzar-se també a la branca màster o qualsevol altra branca i resetejar qualsevol commit indicant el seu dientificador SHA-1. En aquest cas l'hem utilitzat en el cas particular de tornar a l'estat en què fusionem la branca amb la branca principal, però podria ser qualsevol.

    • Pas 8. Eliminant una branca per complet

    Finalment, si el que desitgem és eliminar completament una branca, farem servir l'ordre git branch, amb el paràmetre -d o -D, i indicant-li el nom de branca a eliminar.

    Si la branca que volem eliminar ha estat fusionada amb la branca màster i totes dues són iguals, podem utilitzar el paràmetre -d:

    $ git branch -d proves

Deleted branch proves (was 35d135f).

    D'altra banda, en cas que la branca no estigués fusionada, ens avisarà amb un missatge semblant al següent:

    error: The branch 'proves' is not fully merged

    De manera que si volem descartar realment els canvis sense fussionar les branques, farem servir el paràmetre -D:

    $ git branch -D proves
