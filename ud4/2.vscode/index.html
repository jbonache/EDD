    2. VSCode i Java

    Visual Studio Code i Java

    A la segona unitat vam veure com instal·lar Visual Studio Code, i algunes extensions per a python. Anem a veure com configurar-lo per tal de poder utilitzar-lo en el desenvolupament d'aplicacions Java. Aprofitarem així tota la potència de Java i la lleugeresa de l'editor. En aquest apartat veurem com afegir el suport per a Java a Visual Studio Code, i com configurar-lo perquè utilitze les eines del JDK instal·lat prèviament.

    Extensions per a Java

    Visual Studio Code disposa de centenars d'extensions relacionades amb Java. Per començar, utilitzarem l'extensió Extension Pack for Java. Per a això, fem clic en el botó de l'activitat d'Extensions, la busquem en la barra superior, i fem clic en el botó d'instal·lar:

    Realment, no es tracta d'una extensió en si, sinó d'un conjunt d'extensions, format per:

    • L'extensió Language Support for Java(TM) by Red Hat, amb el suport de Java per a l'IDE Eclipse que ens ofereix ressaltat i comprovació de la sintaxi i errors de compilació, autocompletat de codi, refactorització, suport a JavaDoc, o petits fragments de codi reutilitzable d'ús comú (snippets).
    • L'extensió Project Manager for Java, que es complementa amb l'anterior i proporciona eines d'exploració, creació i construcció de projectes Java.
    • L'extensió Debugger for Java, amb un depurador lleuger que permet llançar les aplicacions, afegir punts de ruptura, pausar i continuar l'execució, seguiment de variables, etcètera.
    • L'extensió Java Test Runner, per a executar diferents proves de funcionament, així com tests unitaris de JUnit, depuració o logs.
    • L'extensió Maven for Java, per a la gestió de projectes Maven, dels quals parlarem en següents apartats. Aquesta extensió ens facilita un explorador de projectes, i proporciona eines per a la seua generació, validació, compilació, realització de tests o desplegament.
    • L'extensió Gradle for Java, per a la gestió de projectes en el sistema Gradle, que també treballarem més endavant.
    • L'extensió Visual Studio IntelliCode, que ofereix algunes característiques de desenvolupament basades en intel·ligència artificial, de manera que és capaç d'inferir què anem a escriure segons el context del codi i l'aprenentatge automàtic a partir del codi ja generat.

    En principi, això hauria de ser suficient per tindre VSCode configurat per treballar amb Java.

    En ocasions, si no s'ha configurat correctament el JAVA_HOME, cal indicar la ruta específicament a través de la propietat "java.jdt.ls.java.home" de la configuració.

    Una altra extensió interessant que podeu utilitzar és la Java Snippets de Seudev:

    Aquesta extensió ofereix una sèrie de snippets o fragments/plantilles de codi d'ús comú, que ens poden agilitzar la creació d'aplicacions. En la documentació a la web o al propi VSCode de l'extensió podeu veure quines són les abreviacions que utilitza per als snippets.

    El nostre primer codi Java en VSCode. Depuració.

    Una vegada instal·lades les extensions i configurat l'entorn, anem a crear el nostre primer codi. Per a això, creem un nou fitxer en la nostra carpeta o espai de treball, que anomenarem Hola.java.

    Si dins el fitxer en blanc premem la lletra c l'extensió de Java Snippets ens suggerirà utilitzar l'snippet per definir una classe, i amb l'abreviació main ens permet generar el mètode main. Finalment, dins aquest main, podem escriure sout, per fer directament un System.out.println.

    Amb tot això fem un primer "Hola Món":

    public class Hola {
    public static void main(String[] args) {
        System.out.println("Hola Món");
    }
}

    Aquest codi podem llançar-lo de dues formes:

    1. A través de la terminal, obrint una nova terminal, i després compilant-lo amb javac per generar el fitxer .class amb el bytecode i executant-lo amb java:
    $ javac Hola.java

$ java Hola

Hola Món
    1. Directament des de l'IDE, amb el botó d'executar, situat a la part superior a la dreta, seleccionant Run Java:

    Si parem atenció també a la finestra on es troba el codi, veurem que sobre la declaració del main tenim també dues opcions Run i Debug que podem fer servir també per llançar el codi i per depurar-lo. Una altra opció per executar el codi és mitjançant la combinació de tecles Ctrl+F5.

    Depuració del codi

    L'opció de Debug inicia la depuració del programa. Això ens permet executar pas a pas el nostre programa, per al que necessitem establir un punt de ruptura o breakpoint. Fer això és bastant senzill. Si ens fixem en l'editor, quan passem el ratolí pel costat esquerre dels números de línia, apareixen uns punts de color taronja. Si fem clic en un d'ells, s'estableix un breakpoint en aquesta línia, de manera que l'execució es pare quan arribe a ell, i ens permeta realitzar una execució pas a pas. No totes les línies de codi són susceptibles de ser punts de ruptura. Per exemple, les línies on obrim o tanquem claus únicament no poden ser-ho, ja que no contenen codi que s'execute.

    Per exemple, podem provar el següent codi que fa un bucle amb un if intern:

    public class Hola {

    public static void main(String[] args) {
        System.out.println("Hola Món");

        for (int i = 0; i < 5; i++) {
            if (i%2==0)
                System.out.println(i+" és parell");
        }
    }

}

    Si afegim un breakpoint en la línia del segon println, i comencem la depuració, veurem que l'execució es para abans de mostrar aquesta línia, i que la barra d'estat ha passat a color taronja, indicant que estem en mode depuració. La interfície haurà canviat lleugerament, mostrant a la part esquerra alguns detalls de l'execució, com el valor de les variables que es troben actives o la pila de crides, entre altres coses:

    Per a continuar amb l'execució fins al final, farem clic en la primera icona blava de la barra de depuració que va aparèixer en la part superior (o premem F5), o si desitgem executar pas a pas, utilitzarem la segona icona (o premem F10). Per a eliminar un breakpoint únicament haurem de tornar a fer clic sobre el cercle taronja que apareix al costat de la línia.
