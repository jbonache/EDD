    Cas Pràctic Ampliat 6: Gradle

    Introducció

    Gradle és una eina moderna que combina la flexibilitat d'Ant amb les capacitats de cicle de vida i proves de Maven. Aquest cas pràctic mostra com utilitzar Gradle per gestionar un projecte, des de la seva creació fins a l'execució.

    Instal·lació

    Descàrrega i configuració manual

    1. Descarregar Gradle des de la seva pàgina oficial:

      • https://gradle.org/install
      • Selecciona només els binaris (gradle-<versió>-bin.zip).

    2. Descomprimir l'arxiu i moure'l al directori d'instal·lació: 

      unzip gradle-<versió>-bin.zip
sudo mv gradle-<versió> /opt/

    3. Afegir Gradle al PATH: 

      export PATH=$PATH:/opt/gradle-<versió>/bin

    4. Comprovar la instal·lació: 

      gradle -v

    Creació del projecte

    Utilització de Gradle Init

    1. Crea una carpeta per al projecte: 

      mkdir HelloGradle && cd HelloGradle

    2. Executa el comandament d'inicialització: 

      gradle init

    Selecciona les opcions següents durant l'assistent: - Tipus de projecte: Application - Llenguatge: Java - Subprojectes: No - DSL: Groovy - Framework de proves: JUnit 4

    Estructura generada: 

    HelloGradle/
├── app/
│   ├── build.gradle
│   ├── src/
│   │   ├── main/
│   │   │   └── java/com/example/App.java
│   │   └── test/
│   │       └── java/com/example/AppTest.java
├── settings.gradle
└── build.gradle

    Configuració de build.gradle

    Modifica el fitxer app/build.gradle per afegir configuracions personalitzades: 

    plugins {
    id 'application'
}
repositories {
    mavenCentral()
}
dependencies {
    implementation 'com.google.guava:guava:30.1.1-jre'
    testImplementation 'junit:junit:4.13.2'
}
application {
    mainClass = 'com.mgh.edd.Hola'
}

    Construcció i execució

    1. Construir el projecte: 

      gradle build

    2. Executar l'aplicació: 

      gradle run --args="Josep"
      Sortida esperada: 
      Hola Josep

    3. Netejar el projecte (elimina build/): 

      gradle clean

    4. Utilitzar el wrapper de Gradle: 

      ./gradlew build
./gradlew run

    Observacions finals

    • Versió de Gradle: Es recomana utilitzar la versió més recent compatible amb el projecte.
    • Resolució d'errors de proves: Si es produeixen errors en AppTest.java, es pot eliminar el directori src/test.
    • Eines de suport: Extensions com Gradle Extension Pack per a VSCode faciliten la gestió de projectes Gradle.
