    1. La plataforma Java

    La plataforma Java

    La plataforma Java és un entorn per al desenvolupament d’aplicacions multiplataforma, que té com a llenguatge de desenvolupament principal el propi llenguatge Java. Com ja comentàrem en la primera unitat, les aplicacions Java són compilades a un llenguatge intermedi o bytecode, que s’executen sobre la màquina virtual de Java (JVM).

    Aquesta màquina virtual, juntament amb les biblioteques de Java, conformen el que és l’entorn d’execució de Java, Java Runtime Environment o JRE, que és l’entorn que necessitarem per a executar qualsevol aplicació Java en un equip.

    Ara bé, quan ens dediquem al desenvolupament d’aplicacions Java, necessitarem un altre component de la plataforma, el JDK, Java Development Kit o Kit de desenvolupament de Java. Aquest component és el que contindrà les eines necessàries per a compilar a bytecode els nostres programes en llenguatge font. En general, a aquest conjunt d’eines de desenvolupament se’ls coneix de forma genèrica com a Software Development Kits (SDK).

    Quan instal·lem Java en el nostre ordinador, a més de la màquina virtual, s’instal·len una sèrie de biblioteques; programari addicional que ens proporciona certes funcionalitats d’ús comú, i que es coneixen com la biblioteca estàndard de Java. Aquestes biblioteques proporcionen una interfície de programació d’aplicacions o API (Application Programming Interface).

    Existeixen, a més, diverses edicions de la plataforma Java. La que utilitzarem habitualment serà l’edició estàndard, JavaSE (Java Standard Edition). Quan parlem habitualment de JDK, solem referir-nos a aquesta edició, que ofereix una API per al desenvolupament d’aplicacions d’escriptori independents i basades en interfícies gràfiques. Per a aplicacions comercials a major escala, com aplicacions web o aplicacions distribuïdes, la plataforma Java compta amb l’edició empresarial Java Enterprise Edition o JavaEE. Aquesta edició està construïda sobre JavaSE, a la qual incorpora una API d’alt nivell. Empreses com Oracle, RedHat i IBM ofereixen implementacions pròpies d’aquesta API. Per altra banda, cal mencionar JavaME, o l’edició Micro, amb una API pensada específicament per al desenvolupament d’aplicacions en dispositius mòbils i integrats.

    Implementacions del JDK

    Java va ser desenvolupat en la dècada dels 90 per Sun Microsystems. L’any 2010, Sun va ser adquirida per Oracle, i amb ella, totes les seues tecnologies, Java inclosa. Abans d’això, en 2007, Sun va llicenciar la majoria de les seues tecnologies sota la Llicència Pública General de GNU (GNU/GPL).

    Així doncs, existeixen actualment dues versions principals del JDK: la mantinguda per Oracle (Oracle JDK) i OpenJDK, la versió lliure, fruit d’aquest esforç de Sun abans de la seua venda a Oracle, mantinguda per la comunitat OpenJDK.

    Respecte al llançament de versions del JDK d’Oracle, des de Java 10 es llancen noves versions cada 6 mesos, amb un període de suport curt, mentre que les versions LTS (Long Term Support), es llancen cada 3 anys. Les últimes versions del JDK d’Oracle són:

    • JavaSE 21 (LTS), Setembre de 2023.
    • JavaSE 22, Març de 2024.
    • JavaSE 23, Octubre de 2024.

    A més, des de Gener de 2019, es requereix d’una llicència d’Oracle per a accedir a les actualitzacions i el suport.

    Per la seua banda, OpenJDK, encara que siga de codi obert, no deixa de ser una implementació oficial de JavaSE. De fet, el JDK d’Oracle es basa en OpenJDK, i igual que aquest, ofereix noves versions cada 6 mesos, corresponent-se el número de versió amb el del JDK d’Oracle.

    A més d’aquestes, existeixen altres implementacions de Java, com Amazon Corretto, IBM JDK o Azul Zulu, entre altres.

    Instal·lació d'OpenJDK

    Instal·lació en sistemes Linux

    OpenJDK es troba disponible en tots els repositoris de programari de les principals distribucions de GNU/Linux. Anem a instal·lar la versió 21 de l’OpenJDK en un sistema basat en Ubuntu. Per a això, podem fer-ho bé des del centre de programari que ofereix la distribució, o mitjançant la línia d'ordres. Utilitzarem aquest segon mètode, per ser més genèric.

    Obrirem una nova terminal, i en primer lloc actualitzem la memòria cau de paquets de programari amb l'ordre (escrivim l'ordre sense el prompt $ i premem intro):

    sudo apt update

    Això ens demanarà la contrasenya d’administrador, i després d’això, descarregarà l’última llista de paquets disponibles per a la nostra distribució.

    Una vegada tinguem la memòria caché de paquets actualitzada, instal·lem la versió LTS més recent del JDK amb:

    sudo apt install default-jdk

    En aquest cas, la versió que instal·la de l’OpenJDK és la 21 (openjdk-21-jdk), que es correspon amb l’última LTS. Fixeu-vos que a part d’aquest, la instal·lació arrossega diversos paquets com a dependències (és a dir, que OpenJDK requereix d’ells per a funcionar). Entre ells es troba el paquet openjdk-21-jre, que és el que implementa el Java Runtime Environment, que com hem dit anteriorment, forma part del JDK.

    En Ubuntu, la instal·lació es realitza per defecte en la carpeta /usr/lib/jvm/java-21-openjdk-amd64. A més, es genera l’enllaç /usr/bin/java que apunta a /etc/alternatives/java, que al seu torn és també un enllaç a /usr/lib/jvm/java-21-openjdk-amd64/bin/java. Els alternatives ens serviran per a utilitzar diversos entorns Java en el nostre ordinador. Igual que s’ha fet amb l'ordre java, que és qui invoca la màquina virtual, s’han generat els enllaços corresponents per al compilador javac. Podem comprovar les versions d'ambdós escrivint en un terminal:

    $ java --version
...
$javac --version

    Si bé amb això ja tenim disponibles des de qualsevol lloc les ordres java i javac, configurarem la variable d'entorn JAVA_HOME, necessària en algunes aplicacions Java, i que conté la carpeta d'instal·lació. A més, actualitzarem la variable del sistema PATH perquè incloga també aquesta carpeta. Per això, a la nostra carpet ad'usuari editaríem el fitxer ~/.bashrc i afegirem al final les següents línies (no ho feu encara!):

    # JAVA_HOME
if [ -d /usr/lib/jvm/java-21-openjdk-amd64/ ]; then
    export JAVA_HOME="/usr/lib/jvm/java-21-openjdk-amd64/"
fi

# Afegim el path dels binaris per a Java
if [ -d "$JAVA_HOME/bin" ] ; then
    PATH=$JAVA_HOME/bin:$PATH
fi

    Amb això establiríem Java 21 com a Java predeterminat. Ara bé, si modifiquem la versió de java predeterminada amb:

    $ sudo update-alternatives --config java

  Selecció    Camí                                        Prioritat  Estat
------------------------------------------------------------
* 0            /usr/lib/jvm/java-21-openjdk-amd64/bin/java   2111      mode automàtic
  1            /usr/lib/jvm/java-17-openjdk-amd64/bin/java   1711      mode manual
  2            /usr/lib/jvm/java-21-openjdk-amd64/bin/java   2111      mode manual

    I seleccionem una versió diferent a la 21, el $JAVA_HOME ja no es modificaría. Una forma diferent de fer-ho seria afegint al ~/.bashrc el següent:

    # Estableix JAVA_HOME basat en l'alternativa seleccionada
export JAVA_HOME=$(dirname $(dirname $(readlink -f $(which java))))

# Afegim el path dels binaris per a Java
export PATH=$JAVA_HOME/bin:$PATH

    Amb això, agafaria el JAVA que hàgem configurat amb l'update-alternatives. De tota manera, i per tal que els canvis tinguen efecte, caldrà fer un source d'aquest fitxer:

    source ~/.bashrc

    O bé obrir una terminal nova.

    Instal·lació a Windows

    Per a la instal·lació i configuració del JDK a Windows començarem descarregant l'OpenJDK des del lloc web de l'arxiu de version: https://jdk.java.net/archive/. Els binaris de la versió 21 estaran a l'apartat 21.0.2 (build 21.0.2+13).

    Des d'aquesta pàgina, descarreguem el binari per a Windows/x64, i el descomprimim des d'on l'hàgem descarregat (botó dret del ratolí, i Extraure-ho tot). Si, per exemple, ho hem descomprimit a C:\java, de manera que ens genera la carpeta per al JDK C:\java\jdk-21.0.1. Podem descomprimir-lo on siga, sempre que recordem aquesta ruta per establir les variables corresponents.

    Per poder accedir al JDK des de qualsevol lloc, cal crear la variable d'entorn JAVA_HOME i actualitzar el PATH, per llançar java des de qualsevol ubicació.

    Per crear JAVA_HOME, haurem d'obrir la finestra de configuració de variables del sistema. Per això, des del menú d'inici, busquem Sistema de Windows > Panell de Control. I des d'aquest panell de Control, Sistema i Seguretat > Seguretat i Manteniment > Configuració avançada del sistema. I a la finestra que apareix, disposem d'un botó per configurar les variables d'entorn. També podrem accedir-hi més fàcilment cercant amb la lupa Editar les variables d'entorn.

    A la finestra per establir les variables, ens situarem a la part de Variables del Sistema, i seleccionarem Nova. El nom d'aquesta nova variable serà JAVA_HOME, i us donarem per valor la ubicació de la carpeta on es va instal·lar el JDK (en el nostre cas C:\java\jdk-21.0.1):

    Ara ens queda modificar la variable PATH. Per això, des de la finestra anterior, busquem la variable Path a les variables del sistema, la seleccionem i fem clic al botó d'Editar. A la finestra d'edició de la variable, seleccionarem l'opció Examinar per cercar el directori bin de la instal·lació del JDK i afegir-lo al Path.

    Donem a acceptar a totes les finestres, i ja tindrem disponible el nostre JDK en tot el sistema. Per exemple, podem obrir una terminal i escriure:

    C:\Users\Alumne> java --version
