    Cas Pràctic Ampliat 5: Maven

    Introducció

    En aquest cas pràctic treballarem amb Maven, una eina essencial per a la construcció d'aplicacions, gestió de dependències i distribució. Apliquem els conceptes teòrics a un projecte pràctic, aprenent a crear, configurar, compilar i empaquetar un projecte.

    Instal·lació de Maven

    En Ubuntu

    1. Actualitzem la llista de paquets: 

      sudo apt update

    2. Instal·lem Maven: 

      sudo apt install maven

    3. Verifiquem la instal·lació: 

      mvn --version

      Exemple de sortida: 

      Apache Maven 3.6.3
Maven home: /usr/share/maven
Java version: 11.0.11
OS name: "linux"

    En Windows

    Consulta l'Annex I del document per obtenir instruccions detallades sobre la instal·lació de Maven en Windows 10.

    Creació del projecte

    Utilitzant Maven Archetypes

    1. Situa't al directori on vols crear el projecte i executa: 

      mvn archetype:generate -DgroupId=com.mgh.edd -DartifactId=saludaMaven     -DarchetypeArtifactId=maven-archetype-quickstart -DarchetypeVersion=1.4 -DinteractiveMode=false

      Explicació dels paràmetres: - groupId: Identificador únic de l'organització (ex. com.mgh.edd). - artifactId: Nom del projecte (ex. saludaMaven). - archetypeArtifactId: Especifica l'arquetip base.

    2. Afegeix els fitxers Hola.java i Saluda.java al directori del projecte: 

      cp {carpeta_saludaBase}/src/com/mgh/edd/*.java saludaMaven/src/main/java/com/mgh/edd/

    3. Elimina el fitxer App.java: 

      rm saludaMaven/src/main/java/com/mgh/edd/App.java

    Estructura del directori resultant: 

    saludaMaven/src/main/java/
└── com
    └── mgh
        └── edd
            ├── Hola.java
            └── Saluda.java

    Configuració del pom.xml

    Modifiquem el fitxer pom.xml per indicar la classe principal al generar l'empaquetat. Afegeix al plugin maven-jar-plugin: 

    <plugin>
    <artifactId>maven-jar-plugin</artifactId>
    <version>3.0.2</version>
    <configuration>
        <archive>
            <manifest>
                <addClasspath>true</addClasspath>
                <mainClass>com.mgh.edd.Hola</mainClass>
            </manifest>
        </archive>
    </configuration>
</plugin>

    Construcció i execució

    1. Compilació del projecte: 

      mvn compile

    2. Execució de l'aplicació (amb arguments): 

      java -cp target/classes com.mgh.edd.Hola Josep
      Sortida esperada: 
      Hola Josep

    3. Empaquetament en un arxiu JAR: 

      mvn package

    4. Execució del JAR: 

      java -jar target/saludaMaven-1.0-SNAPSHOT.jar Josep

    Observacions finals

    • És fonamental garantir que Maven i Java estan correctament configurats.
    • Sempre és recomanable revisar el fitxer pom.xml per assegurar que les dependències i plugins són actualitzats.
